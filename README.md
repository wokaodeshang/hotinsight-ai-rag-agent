# HotInsight：面向搜索场景的 AI 热点消费助手

HotInsight 是一个用于产品实习作品集展示的 Web Demo。它不是普通热搜榜，而是一个围绕“发现热点、理解背景、判断争议、继续搜索、关注后续、反馈偏好”的热点搜索消费决策系统原型。

## 项目背景

作者具备人工智能硕士背景，并有 RAG、Agent 意图路由、Prompt 优化、BadCase 分析和用户 Query 分类经验。本项目用于展示如何把 AI 能力迁移到热点消费、搜索推荐和信息理解场景。

## 产品目标

- 帮助用户在看到热点后，用 30 秒理解事件背景与当前进展。
- 展示时间线、争议焦点和相关搜索建议，降低碎片化信息带来的理解成本。
- 模拟 AI 热点问答，让用户通过预设问题继续追问背景、原因、进展和搜索扩展。
- 通过“关注后续、不感兴趣、总结有用、总结没用”等反馈信号影响推荐策略。
- 用数据看板和策略页展示产品分析、意图路由和推荐策略设计能力。

## 功能模块

- `/` 今日百度热搜总览：优先从百度热搜实时榜抓取 Top 热点，展示排名、热搜指数、类别标注、用户意图、搜索承接动作和风险信号；抓取失败时回退到 2026-07-03 快照。
- `/topic/[id]` 热点详情：产品解读、生命周期、信息可信度、指标解释、30 秒看懂、事件时间线、争议焦点、相关搜索、AI 推荐追问和互动反馈。
- `/ask` AI 热点问答：左侧选择热点，右侧模拟结构化 AI 回答和意图识别结果。
- `/dashboard` 数据分析看板：类别数量、热度趋势、情绪分布、搜索意图 Top 5、增长速度 Top 5，并输出产品结论和下一步动作。
- `/strategy` 意图路由与推荐策略：用户路径、意图 Schema、路由流程、推荐公式和 BadCase 归因。
- `/research` 调研与竞品分析：用户痛点、用户需求、核心机会判断、竞品分析、机会点、功能映射和验证指标。

## 数据说明

首页真实热搜数据逻辑位于 `src/lib/baiduHot.ts`，优先读取百度热搜实时榜，并用 `src/data/baiduHotSnapshot.ts` 作为兜底快照。其余详情、问答、看板和策略页面的核心 mock 数据位于 `src/data/hotTopics.ts`，用于展示 AI 摘要、意图路由、时间线、争议焦点、相关搜索、推荐追问、模拟回答和反馈指标。`src/lib/productInsights.ts` 会基于 mock 指标推导热点生命周期、信息可信度、主用户意图、推荐理由、承接动作和风险信号。

## 作品集展示价值

这个 Demo 展示的重点不是页面堆砌，而是把 AI 产品能力转化为搜索体验优化：

- RAG 思路：把热点摘要、时间线和观点作为可检索上下文。
- Agent 意图路由：把用户点击和提问映射到背景理解、原因追问、进展追踪、观点判断、实用关联、搜索扩展和情绪反馈。
- Prompt 与回答结构：模拟包含背景、原因和后续搜索建议的结构化回答。
- BadCase 分析：把“总结过泛、搜索不准、分类错误、重复推荐”等问题转化为可迭代策略。
- 推荐策略：用热度、增长、兴趣匹配、搜索意图和负反馈构建推荐优先级公式。`n- 指标意识：把“热度、增长、争议、搜索意图、负反馈”解释为产品动作，而不是只展示假数字。`n- 作品表达：明确说明 mock 数据用于策略演示，重点展示热点产品的信息架构、AI 意图路由和反馈迭代闭环。
