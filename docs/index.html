Portfolio Project · AI Product Prototype

面向搜索场景的 AI 热点消费助手

HotInsight 不是再做一个热搜榜，而是把“看到热点”之后的理解、判断、搜索、追问、关注后续和反馈偏好，设计成一套可迭代的产品链路。

RAG 摘要Agent 意图路由Query 分类BadCase 归因推荐策略

完整可交互 Demo 部署在 Vercel。部分大陆网络访问 Vercel 可能较慢或需要代理；本页部署在 GitHub Pages，用于快速展示项目背景、产品方法和核心价值。

问题定义

传统热榜解决“发现热点”，但没有很好解决“为什么重要、我该怎么看、下一步搜什么、我的反馈是否影响推荐”。HotInsight 的机会是补齐热榜和搜索之间的 AI 理解层。

标题信息密度低

用户看到热搜标题后，仍需要补背景、时间线和影响范围。

观点和事实混杂

争议热点容易被情绪放大，需要分层展示事实、立场和信息缺口。

搜索承接不足

看完热点后用户会继续搜原因、进展、影响和行动建议，但缺少结构化 Query 引导。

产品路径

将热点消费拆成可观察、可路由、可反馈的连续链路。

方法框架

意图 Schema

背景理解、原因追问、进展追踪、观点判断、实用关联、搜索扩展、情绪反馈。

模块路由

根据点击和提问匹配 30 秒摘要、时间线、争议焦点、相关搜索、AI 追问和反馈面板。

反馈迭代

关注后续、不感兴趣、总结有用、总结没用进入推荐权重和 BadCase 归因。

推荐分 = 热度分 * 0.35 + 增长速度 * 0.25 + 用户兴趣匹配 * 0.20 + 搜索意图强度 * 0.15 - 负反馈 * 0.05

作品集展示价值

AI 能力迁移

把 RAG、Agent 路由、Prompt 结构化、BadCase 分析和 Query 分类经验迁移到热点消费场景。

产品实习匹配

覆盖热点产品 JD 中的功能设计、策略迭代、数据分析、自主调研、竞品分析和 AI 产品探索。
